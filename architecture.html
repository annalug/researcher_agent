02 — Agentes

Detalhes dos Agentes

Cada agente possui um system prompt especializado, acesso a ferramentas específicas, e opera de forma independente dentro do grafo.

🧠

Supervisor

Roteador Central

Analisa a solicitação do usuário e decide qual agente especialista deve responder. Retorna apenas o nome do agente: researcher, critic, editor ou direct.

  • classificação de intenção
  • roteamento condicional
🔍

Researcher

Agente Pesquisador

Busca papers em múltiplas fontes, sintetiza resultados com métricas de impacto, identifica autores-chave e sugere artigos altamente citados e recentes.

  • search_arxiv
  • search_semantic_scholar
  • get_arxiv_paper_details
  • get_paper_citations
  • fetch_paper_from_url
🔬

Critic

Agente Crítico

Revisa drafts científicos avaliando contribuição, cobertura da literatura, metodologia, resultados, limitações, estrutura e linguagem acadêmica.

  • search_arxiv
  • search_semantic_scholar
  • análise de draft
✏️

Editor

Editor Científico

Especializado em formatação e estilo de revistas top (IEEE, Nature, ACM, Elsevier). Corrige gramática, melhora argumentação e formata citações.

  • formatação IEEE/APA
  • revisão gramatical
  • melhoria de abstract
03 — Estado

AgentState

O estado compartilhado entre todos os nós do grafo. Definido como um TypedDict do LangGraph com redução de mensagens via add_messages.

Campo Tipo Descrição
messages list[BaseMessage] Histórico de mensagens com redução automática via add_messages. Inclui HumanMessage e AIMessage.
next_agent str Nome do próximo agente a ser acionado. Definido pelo Supervisor: researcher, critic, editor ou direct.
draft_text str Texto do rascunho do usuário para análise pelo Critic ou Editor. Pode incluir texto de PDF extraído.
research_context str Papers e resultados de busca acumulados pelo Researcher. Truncado em 3000 caracteres para contexto entre turnos.
iteration int Contador de iterações para evitar loops infinitos no grafo.
04 — Ferramentas

Search Tools

Ferramentas de busca acadêmica implementadas como @tool do LangChain, acessíveis via function calling do LLM.

search_arxiv()

ArXiv API

Busca papers por relevância. Retorna título, autores, resumo, link e PDF para até N resultados.

get_arxiv_paper_details()

ArXiv API

Detalhes completos de um paper pelo ID: categorias, DOI, comentários e resumo integral.

search_semantic_scholar()

Semantic Scholar API

Busca com métricas ricas: contagem de citações, PDF open access, filtro por ano.

get_paper_citations()

Semantic Scholar API

Retorna papers que citam um artigo específico, com autores e contagem de citações.

extract_pdf_text()

PyMuPDF (fitz)

Converte PDF local para Markdown legível. Trunca em 15k caracteres para caber no contexto do LLM.

fetch_paper_from_url()

HTTP + BeautifulSoup

Busca conteúdo de URL (HTML ou PDF online) e converte para texto limpo.
